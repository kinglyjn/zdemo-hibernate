
hibernate提供了以下几种检索对象的方式：
	1. OID检索方式：按照对象的oid来检索对象
	2. 导航对象图检索方式：根据已经加载的对象导航到其他对象
	3. HQL检索方式：使用面向对象的HQL查询语言
	4. QBC检索方式：使用QBC（query by criteria）API来检索对象，这种API封装了基于字符串形式的查询语句，
				   提供了更加面向对象的查询接口。
	5. 本地SQL检索方式：使用本地数据库的SQL查询语句。
	
				   
HQL检索方式：
	他是面向对象的查询语言，在形式上和sql有些相似，在hibernate中是使用最广泛的一种检索方式。有如功能：
		1. 在查询语句中设置各种查询条件（where）
		2. 支持投影查询（即仅检索出对象的部分属性）
		3. 支持分页查询
		4. 支持连接查询
		5. 支持分组查询，允许使用having和group by关键字
		6. 提供内置聚集函数，如sum()|min()|max()等
		7. 支持子查询
		8. 支持动态绑定参数
		9. 支持排序（orderby关键字）
		10. 能够调用用户自定义的sql函数或标准的sql函数
	
	
	HQL绑定参数：
		机制依赖于JDBC API的PreparedStatement的预定义sql语句功能
		形式有两种：
			1.按参数名称绑定: 关键符号为":"
			2.按参数位置绑定: 关键符号为"?"
		相关方法：
			setEntity(xx) 把参数和一个持久化类绑定，实体只为id赋值即可
			setParameter(xx) 绑定一个任意类型的参数，该方法的第三个参数显示地指定hibernate映射类型
			
	
	分页查询：
		setFirstResult(int) 设定从哪一个对象开始检索，参数firstResult表示这个对象在查询结果中的索引位置，索引位置
							的起始值为0，默认情况下query从查询结果中的第一个对象开始检索。
		setMaxResults(int) 设定一次最多检索出的对象的数目，在默认情况下，query和criteria接口检索出查询结果中所有的
							对象。
							
	
	迫切左外连接
		*left join fetch关键字表示迫切左外连接策略。
		*list()方法返回的集合中存放实体对象的引用，每个Dept对象关联的Emp集合都被初始化，存放所有关联的Emp对象，
		 查询结果中可能包含重复元素，可以通过一个HashSet来过滤重复元素。
		 
		 测试迫切左外连接（加了fetch关键字，返回Dept实体组成的list集合）（推荐使用！）
		 去重方式：1.使用distinct关键字
	 			  2.使用Set集合过滤
								
		 测试左外连接（返回数组组成的list）
	 	 去重方式：只能通过使用distinct关键字去重
	 	 
	 	*如果在hql中没有显式指定检索策略，将使用映射文件配置的检索策略。
	 	*如果hsql中显式指定了检索策略，hql会忽略映射文件中设置的迫切左外连接策略，所以如果希望hql采用
	 	 迫切左外连接策略，就必须在hql查询语句中显式地指定它。
